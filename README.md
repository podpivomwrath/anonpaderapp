# Текстовая MMORPG-бот для ВКонтакте

vkbottle + Callback API (aiohttp), PostgreSQL (SQLAlchemy 2.0 async + Alembic), Redis, APScheduler. Реализована игровая логика по согласованному дизайну v2: формулы боя, два боевых движка (тиковый и дуэльный), подкласс-референс Страж, биржа-дилер, смерть/респавн, ставки PvP, заточка/пробуждение.

## Требования

- Python 3.12+
- Docker (для Postgres и Redis)

## Быстрый старт

```bash
# 1. Поднять Postgres и Redis
docker compose up -d

# 2. Виртуальное окружение и зависимости
python -m venv .venv
.venv\Scripts\activate        # Windows
# source .venv/bin/activate   # Linux/macOS
pip install -r requirements.txt

# 3. Конфиг
copy .env.example .env        # Windows (cp на Linux/macOS)
# заполнить VK_TOKEN, VK_GROUP_ID, VK_CONFIRMATION_CODE, VK_SECRET (см. ниже)

# 4. Миграции
alembic upgrade head

# 5. Запуск бота
python main.py
```

Сервер поднимется на `http://0.0.0.0:8080`, вебхук — `POST /vk/callback`, health-check — `GET /health`.

## Настройка Callback API в VK

1. Создай сообщество (или используй существующее) → **Управление → Настройки → Работа с API**.
2. Вкладка **Ключи доступа** → создать ключ с правами на сообщения → в `.env` как `VK_TOKEN`. ID сообщества → `VK_GROUP_ID`.
3. Вкладка **Callback API**:
   - **Адрес** — публичный URL сервера + `/vk/callback`. Для локальной разработки нужен туннель (например, `ngrok http 8080` или cloudpub) — VK должен достучаться до сервера снаружи.
   - Поле **«Строка, которую должен вернуть сервер»** → в `.env` как `VK_CONFIRMATION_CODE`.
   - **Секретный ключ** → в `.env` как `VK_SECRET` (рекомендуется).
   - Запусти `python main.py` и нажми **«Подтвердить»** — сервер ответит строкой подтверждения.
4. Вкладка **Типы событий** → включи **«Входящее сообщение»** (message_new).
5. В настройках сообщества включи **Сообщения** (и «Возможности ботов», чтобы работали клавиатуры).

## Что уже работает

- `POST /vk/callback` — подтверждение сервера, проверка секрета, роутинг событий в vkbottle;
- **сюжетный онбординг** (`/start`): сцена пробуждения и весь диалог ведёт NPC **Хранитель Списков** (реплики — `content/npc/list_keeper.json`, не хардкод) → никнейм (валидация формата, регистронезависимая уникальность, фильтр `content/banned_words.txt`) → «проверка крови» / выбор класса (разные стартовые статы, 85 очков) → «четыре дороги» / выбор региона (постоянный выбор). FSM на vkbottle BaseStateGroup, прогресс в БД (`characters.creation_state`) — переживает рестарт бота; `/profile` — профиль с регионом;
- **формулы v2** (`game/combat/formulas.py`) — HP/урон/крит/митигация/резист контролю/сила саппорта; все константы в `game/combat/balance_config.py`;
- **тиковый движок** (`tick_engine.py`) — PvE (без таймера, ждёт всех живых игроков) и групповой PvP (окно 1 мин + досрочный резолв, когда объявили все). Одновременный резолв: защита → урон, мобы ходят после игроков (таунт форсит цель);
- **дуэльный движок** (`duel_engine.py`) — последовательные ходы: рандом первого, жёсткое чередование, второй видит результат, таймер 1 мин → пропуск хода, ничья валидна;
- **Страж** — референсный подкласс (Блок, Провокация PvE-форс/PvP-дебаф, Групповой щит); **Кровавый рыцарь** (лайфстил с капом 8% maxHP/тик), **Отравитель** (яд от статов: 0.6×WIL+0.4×AGI, стаки до 3) и **Тёмный мистик** (Кровавый пакт ×0.76 + конверсия в хил) — базовые действия по откалиброванным значениям патча балансировки; Клинок теней и Элементалист — структура + TODO;
- **биржа-дилер** (`game/economy/exchange.py`) — линейный шаг цены за блок 100 доната, фикс. спред (round-trip убыточен), live-объём в Redis;
- **сервисы**: смерть/респавн (-20% опыта, 1→30 мин), пресеты баффов (3-5, обязательна оборона/утилити), респек 3 уровней, ставки PvP (% из конфига), заточка ±1 до +20 и пробуждение (30% риск уничтожения, слот самоцвета);
- формат % боя (п.11): PvP — целые, рейды — 0.1%, без ложных 0%/100%, health-bar;
- **мир**: координатная сетка -50..50 (Монолит в 0;0, 4 города по углам), перемещение по клеткам (10 сек/клетка, компас-стрелки без букв сторон света), сложность зоны по расстоянию Чебышёва; **города** — меню Наставник/Рынок(заглушка)/За ворота/Отдых; **наставники** 4 регионов (`content/npc/mentors.json`) выдают единый первый квест «убей 10» (наград пока нет — только похвала, `world-patch-1`); за воротами — интерфейс карты, клетка исследуется кнопкой (5-10 сек, ~50% шанс атмосферного фрагмента Пепельной Песни/замечания перед мобом); мобы стартового кольца — бестиарий `content/mobs/starter_ring.json` (порождения раскола, флейвор перед боем), уровень моба = clamp(уровень игрока, зона);
- **навыки классов** (`content/skills/base_skills.json`, combat-patch-2): по 3 на класс с 1 уровня, без маны, баланс через КД в ходах боя (сбрасывается между боями); эффекты — оглушение/заморозка (резист от WIL, контроль срабатывает в ТОТ ЖЕ ход), баффы/дебаффы урона, двойной удар, гарант-крит, уклонение; кнопки навыков с КД-счётчиком; **защита от чейн-контроля** (DR): 3-й контроль подряд урезан ×0.5 + иммунитет на 2 хода;
- **прогрессия** (progression-patch-4): опыт за мобов `15+6×ур` и крупная разовая награда за квест (500), кривая уровней со степенным ростом до 50 и линейной «полкой» после (явный излом); левелап копит очки характеристик (трата — вкладка «Характеристики» в мини-аппе, см. ниже), лорные сообщения; штраф смерти −20% опыта текущего уровня (без понижения уровня); все коэффициенты в `balance_config`;
- **отдых** (8-12 сек вне боя → полное HP) и **авто-респавн** (по таймеру, один батч-сканер мёртвых, живой отсчёт через edit, флаг `RESPAWN_LIVE_COUNTDOWN`); HP персистентно между боями (`characters.current_hp`); чистка кнопок в моменты ожидания;
- **атмосферный слой** (atmosphere-patch-3): вариативные боевые логи (`content/flavor/combat_log.json`), тексты переходов/отдыха/смерти/респавна (`system.json`), описания локаций по регионам (`locations.json`), Пепельная Песнь и замечания при исследовании;
- **мини-апп «Персонаж»** (`miniapp/`, см. [miniapp/README.md](miniapp/README.md)) — VK Mini App с рабочей вкладкой «Характеристики» (распределение очков, живой предпросмотр производных характеристик) и заглушками Инвентарь/Пресеты/Биржа; авторизация — проверка подписи launch-параметров VK на каждом запросе `/api/miniapp/*` (`bot/miniapp_auth.py`), эндпоинты — `bot/miniapp_api.py`; кнопка «🎭 Персонаж» в боте появляется, как только заполнен `VK_MINIAPP_URL`.

## Структура

```
miniapp/        VK Mini App (React/Vite/VKUI) — хаб персонажа, см. miniapp/README.md
bot/            вебхук aiohttp, хендлеры команд (onboarding/world/combat/basic/fallback), клавиатуры,
                miniapp_auth.py (проверка подписи), miniapp_api.py (/api/miniapp/*)
game/combat/    balance_config, formulas (v2), display, session (runtime),
                skills, resolver, tick_engine (pve/pvp_group), duel_engine
game/classes/   guardian (реализован) + 5 подклассов структурно
game/economy/   exchange.py — дилерская биржа
game/world/     world_config, grid (сетка/зоны), encounters (спавн мобов), travel_scheduler
models/         SQLAlchemy-модели (16 таблиц)
services/       db, user, wallet, death, preset, respec, stake, upgrade, onboarding,
                movement, quest, experience, encounter
content/        JSON-контент: мобы (в т.ч. региональные), предметы, баффы, квесты,
                npc/ (Хранитель Списков, наставники)
migrations/     Alembic: 0001 (скелет) → 0002 (дизайн v2) → 0003 (онбординг) → 0004 (мир/квесты)
tests/          pytest + pytest-asyncio (SQLite in-memory для сервисов)
scripts/wipe.py дев-утилита: полный вайп персонажей (не трогает users), запускать после
                каждого обновления на стадии разработки
```
